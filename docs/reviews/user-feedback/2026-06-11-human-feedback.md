# Human Feedback
- Date: 2026-06-11 14:00

본 문서는 사용자가 웹 브라우저를 이용해 실제 테스트한 결과이다.

## 1. 대시보드 페이지

### 1-1. 최상단 UI
- 해상도에 따라 인사말이 망가짐. 인사말 / 업무 현황 병렬로 구성된 것을 상/하로 변경하는걸 추천함.![alt text](image.png) 이미지 참조.

### 1-2. Inbox 기능
- Inbox 추가 시 미리 Input Type을 선택할 수 없는것이 매우 불편함. Teams/구두 등 선택 할 수 있어야 함.
- Inbox Box에서 프로젝트를 변경할 수 없음. Drapdown 형태로 변경할 수 있어야 함.
- Inbox Box에서 Input type을 변경할 수 없음. Drapdown 형태로 변경할 수 있어야 함.
- Inbox에서 우선순위는 잘 사용하지 않음. 우선순위 기능은 기능을 삭제하기보다, Settings에서 우선순위 기능의 활성화 여부에 따라 선택적으로 기능이 나타나면 좋겠음.
- Inbox box에서 업무 내용을 변경할 수 없음. 텍스트 박스를 수정하거나 수정 버튼을 누르면 Modal이 나타나 각종 속성을 변경할 수 있으면 좋겠음.
- Inbox의 데이터를 삭제할 수 있는 기능이 대시보드에 없음. 오타로 잘못 등록했을 때, 삭제 후 재등록 하는게 자연스러움.
- Inbox에 등록 후 별도로 시간을 지정해야하는 것이 번거로움. 기본값은 현재 시각이 자동으로 들어가면 좋겠음.

### 1-2. 오늘/기한초과/대기 기능
- 이 기능이 반드시 필요한지는 아직 잘 모르겠음. 평가 보류

### 1-3. 프로젝트와 작업 기능
- 칸반을 전체/프로젝트로 선택적으로 볼 수 있는건 Dropdown이 아니라 Toggle이면 좋겠음.
- 프로젝트를 선택했을 때, 선택된 1개의 프로젝트만 칸반으로 보여주는것이 아니라, 각 프로젝트별 독립 칸반을 보여주면 좋겠음. 프로젝트 필터 기능은 지원하면 좋겠음.
- 칸반에서 Hold -> Done으로 이동 불가. Cancled -> Open으로만 이동가능. 의도된 동작인지 확인 필요.

## 2. Inbox 페이지
- Inbox의 히스토리가 필요한 지 잘 모르겠음. Inbox가 너무 복잡하게 느껴짐. 어떠한 경우에 아카이브로 이동시킬지 잘 모르겠음.
- 대시보드에서와 마찬가지로 삭제 기능이 없음.
- 미처리와 전환됨을 구분한 건 좋은 것 같음.
- 태그 기능이 있으나, 어떻게 사용해야 할 지 잘 모르겠음. (아직 미구현이라 평가 보류.)
- 결론적으로 Inbox 페이지는 미처리항목과 전환 완료 항목을 보는 곳으로 사용하면 좋겠음. 사용자가 여기에서 주로 머무는것은 피곤할 것 같음.

## 3. 프로젝트 페이지

- 프로젝트는 최초 생성 시 입력할 정보가 많은데, Quick insert로 생성하는건 조금 불편해보임. Modal 형태가 좋지 않을까?
- 각 프로젝트가 Card view로 구현되었는데, 이게 적절한 방법인지 연구해 볼 필요가 있음. 카드뷰는 데이터 제공량이 너무 작다고 생각함. 아니면 카드를 더 크게 하면 조금 개선되려나?
- 프로젝트의 상태 속성은 Active/on hold / done / archived로 4개가 있는데, 프로젝트 페이지에서는 활성/보관만 탭으로 분리되어있음. 전체/활성/홀딩/완료/보관을 모두 개별 탭으로 보여주는게 좋을까? 아니면 활성탭에서 섹션을 활성/홀딩/완료로 계층 분리해서 나타내는게 좋을까?
- 프로젝트 카드를 누르면 프로젝트 페이지로 들어갈 것이라 기대했는데, 정확하게 프로젝트 이름 텍스트를 눌러야만 진입된다.
- (치명적)프로젝트의 상태를 변경할 수 있는 기능이 없음.

### 3-1. 활성 프로젝트 페이지
- 프로젝트의 데이터가 너무 빈약함. 프로젝트라는 데이터 구조 자체를 고민해봐야 할듯. 특히 초기 설계 단계에서 고려한 각종 링크의 목적이 제대로 반영되지 않은 것 같음.
- 링크의 목적은 관련 데이터가 어디서 관리되는지 보여주기 위함이었음. 예를들자면 다음과 같음.
  - 링크 1: Sharepoint | SHAREPOINT_URL
  - 링크 2: CludDisk | CloudDisk\Project\GM
  - 링크 3: Jira | JIRA_URL
- 종합적으로 활성 프로젝트 페이지는 대대적으로 개편해야 할 것 같음. UI/UX가 와닿지 않음.

## 4. 작업 페이지
- 작업 페이지는 왜 존재하는거지? Inbox와 연결되지 않은 별도의 작업을 만들기 위한건가? 다른 데이터들이랑 뭔가 중복되어서 보여지는 기분이 든다.

## 5. 칸반 페이지
- 대시보드에 이미 칸반이 있는데, 칸반 페이지를 별도로 제공하는 이유가 뭐지? 대시보드에서 보여지는것과 차별점이 있나? 오히려 필터 기능같은건도 없어서 대시보드 페이지보다 별로인데?
- 칸반 페이지도 완전 제거하는걸 고려해봐.

## 6. 결정사항 페이지
- 이 페이지는 왜 존재하지? 아직 결정사항이 제대로 구현이 안돼서 평가하기는 조심스럽다.

## 7. 설정 페이지
- 회의록 템플릿 설정 부분이 좀 고민이다. 현재 회의록의 RAW Data를 Markdown으로 저장하는건 내 철학이 맞다. 근데 입력도 사용자가 Markdown으로 하는게 맞는가? 회의록이란게 템플릿이 크게 변동되지 않을텐데, 미리 만들어진 Form을 사용자가 텍스트 박스만 채워넣는 형태가 좋지 않을까? 이건 심도있게 의견을 따져봐야해.

> 만약 회의록에서 Markdown 직접 작성을 폐기하더라도 Markdown Live Editor Core 기능은 유지하는게 좋겠어. 나중에 활용할 수 있으니까.

## 8. Navigation Bar
- 프로젝트 아래의 선택 구조가 일반적이지 않음. Navigation bar는 일반적으로 fold/unfold로 계층 분리를 하는데, 현재는 활성 프로젝트의 하위 항목을 모두 노출함. 고민 필요함. 다만 현재 구조가 좀 창의적인 면도 있어서 완전 나쁘다고는 할 수 없음.

## 9. 기타
- 각종 변동사항을 로그로 남기는 기능이 있으면 어떨까? 예를들어 언제 Inbox에 데이터가 추가되었고, 언제 삭제되었고, 언제 Task로 할당됐고 등..
